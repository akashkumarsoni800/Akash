# Requirements: School Management System (SMS)

**Defined:** 2026-03-25
**Core Value:** Secure, role-based school management on the Internet Computer.

## v1 Requirements

### Attendance Management
- [ ] **ATT-01**: Teacher can view list of students in assigned classes.
- [ ] **ATT-02**: Teacher can mark daily attendance (Present/Absent).
- [ ] **ATT-03**: Admin can view attendance reports for any class/student.
- [ ] **ATT-04**: Student can view personal attendance history.

### Exam & Result Management
- [ ] **EXAM-01**: Admin can create and schedule exams for subjects.
- [ ] **EXAM-02**: Teacher can enter marks for students in assigned subjects.
- [ ] **EXAM-03**: System automatically calculates grades (A, B, C, D, F) from marks.
- [ ] **EXAM-04**: Student can view personal exam results.

### Dashboard & UI
- [ ] **UI-01**: Role-specific dashboards (Admin, Teacher, Student) with relevant widgets.
- [ ] **UI-02**: Responsive design for mobile and desktop access.
- [ ] **UI-03**: Real-time feedback for data submission and error handling.

## v2 Requirements
- **NOTF-01**: Push notifications for new notices or exam results.
- **COMM-01**: Simple messaging system between teachers and parents/students.
- **DOCS-01**: Document upload for student certificates and IDs.

## Out of Scope
| Feature | Reason |
|---------|--------|
| Fee Management | Not part of current milestone. |
| Timetable Generator | High complexity, manual scheduling used for now. |
